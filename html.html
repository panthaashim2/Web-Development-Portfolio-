HTML: Structuring the Web

HyperText Markup Language (HTML) defines the meaning and structure of web content. It forms the foundation upon which styling (CSS) and behaviour (JavaScript) are added.

What is HTML?

HTML is the most basic building block of the web. It defines the meaning and structure of documents using elements and tags. Other technologies are used to describe a page’s appearance (CSS) or behaviour (JavaScript), but HTML gives semantic meaning to text, images and media so they can be interpreted by browsers and assistive technologies.

Semantic markup improves accessibility and SEO. Headings (<h1>–<h6>) establish document structure; lists (<ul>, <ol>) group related items; links (<a>) connect pages; and forms collect user input. Proper use of these elements helps both humans and machines understand your content.

Table Example

HTML Elements and Their Purpose
Element Description
<header> Represents introductory content or a set of navigational links.
<nav> Contains navigation links for the current document or section.
<main> Denotes the main content unique to the document.
<footer> Holds information about the author, copyright, contact info, etc.

Definition List

Element
An individual component of a web page defined by HTML tags.
Attribute
A modifier on an element that provides additional information, such as class or href.
Semantic
Conveying meaning through markup that reflects the role or purpose of the content.

How It Works

The semantic layout groups related content using elements that convey meaning. Using <article> signals that the block is a self‑contained composition; <section> identifies thematic groupings; and <aside> contains complementary information. <figure> and <figcaption> associate images with captions. In the form, each <label> uses the for attribute to link to its corresponding control by ID, ensuring screen reader users know what each field is for. The required attribute tells the browser to enforce input before submission.

Key Takeaways

  • HTML gives structure and meaning to web pages with semantic elements.
  • Use headings, sections and lists to organise content logically.
  • Accessible forms require labels tied to inputs and appropriate input types.
  • Tables and definition lists provide structured data and term definitions.
